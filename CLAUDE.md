# CLAUDE.md — «Бизнес в кармане» (PocketBiz)

Операционная система для самозанятых (бьюти, репетиторы, частные специалисты): нативное приложение для мастера (iOS/Android) + веб-витрина самозаписи для клиентов + API + Telegram-бот как канал уведомлений.

## Продуктовые аксиомы (не нарушать)
- Приложение ставит ТОЛЬКО мастер. Клиент записывается через веб-витрину/Telegram без установки чего-либо.
- Аудитория без тех-компетенций: минимум полей на экране, всё на русском, mobile-first.
- MVP узкий и рабочий. Никакого «золотого литья»: фича вне текущей фазы = TODO-комментарий, не реализация.

## Структура монорепо (pnpm workspaces + Turborepo)
- `apps/mobile` — Expo (React Native + TypeScript), expo-router, TanStack Query (+persist), Zustand, react-hook-form + zod, expo-notifications, Reanimated. Сборки через EAS.
- `apps/web-booking` — Next.js (App Router), SSR; работает и как сайт `book.<domain>/<slug>`, и как Telegram Mini App (детект по initData).
- `apps/api` — NestJS, Prisma + PostgreSQL, Redis + BullMQ; Telegram-бот (grammY) — модуль внутри API.
- `packages/shared` — типы, zod-схемы, движок расчёта слотов, API-контракт (ts-rest).

## Команды
```bash
pnpm i                      # установка
docker compose up -d        # postgres + redis
pnpm db:migrate && pnpm db:seed
pnpm dev                    # api + web-booking
pnpm dev:mobile             # expo start
pnpm test                   # unit
pnpm lint && pnpm typecheck # обязательны перед каждым коммитом
```

## Жёсткие правила кода
- TypeScript strict, без `any` (исключение — обоснованный комментарий).
- Каждый эндпоинт: zod-валидация входа, rate limiting, ошибки в едином формате `{ code, message }`.
- Вся бизнес-логика слотов/таймзон — только в `packages/shared`, покрыта unit-тестами. Время в БД — UTC; таймзона мастера — поле User.
- Платёжные вебхуки идемпотентны (уникальный `provider_ref`), подпись проверяется, обработка retry-safe.
- Деньги — integer в копейках, никогда float.
- Секреты только через .env / EAS Secrets; `.env.example` поддерживается актуальным. Логи без персональных данных.

## Авторизация
- Мастер: телефон + OTP (интерфейс `SmsProvider`; dev-заглушка пишет код в лог). JWT access 15 мин + refresh с ротацией, хранение в SecureStore.
- Клиент витрины: без регистрации (имя + телефон + чекбокс согласия при записи).
- Серверу не доверять никаким данным клиента без валидации.

## Платежи — два контура, не смешивать
1. Оплата услуг мастера клиентом (офлайн-услуги) → ЮKassa СБП/карты на веб-витрине, вне IAP (Apple 3.1.3(e)). Абстракция `PaymentProvider`, заглушки под Точку/Т-Бизнес/CloudPayments/Stripe.
2. Pro-подписка мастера (цифровая) → ТОЛЬКО RevenueCat/IAP. Источник истины — сервер (вебхуки RevenueCat → Subscription).
- Telegram Stars не используем.

## Уведомления
Клиенту: Telegram-бот (если есть telegram_id) → SMS (лимитировать) → e-mail. Мастеру: всегда push. Все отложенные задания — BullMQ delayed jobs с защитой от дублей (jobId детерминированный).
